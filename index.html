Rule Specificity Breakdown

Ranking from least specific to most specific:
  1. Universal Selector (*)
  2. Type Selector (Element Selector) and Pseudo-elements
  3. Class Selector, Attribute Selector, and Pseudo-classes
  4. ID Selector
  5. Inline Styles
  6. !important

Cascading: If two rules are identical, the one at the bottom of the css page will have higher priority because it is the last to be applied.

Specificity: Overrides all other declarations, regardless of their specificity.

Using !important forces the style to be applied, ignoring all other specificity rules. However, !important can be overridden by another !important with higher specificity.

Useful Tip!
If you open the page on chrome, right-click, and click 'inspect'. It gives you the CSS breakdown and shows you why some rules are beiing overridden, and the numerical value of the CSS. More on CSS numerical breakdown here.

Chapter 2: Selectors

Element Selectors

Elements are like body, p, h1, etc and it automatically selects all those elements. An interesting one is size in px because even with headers it still increases everything proportionately.

Class Selectors

These allow you to get more specific. In CSS you create a class with .[text] {style}, then you can apply to the part of the website you want. So in this example I have applied to the h1 and img within the div.

ID Selectors

This is written in the same way in the stylesheet although with a #. These should rarely if ever be used!! He has stressed it is bad practice to use these and if you do they should only be used once. I get the sense at this stage that you basically want to style everything with class selectors with the occasional element selector.

Grouping Selectors

this is really simple, if you to apply the same style to more than one element selector you simply write a , between the two elements that you wish to apply both to. H1 and H2 are used in this example.

Selectors within Selectors (I think)

Similar to the above, if you remove the comma this time it will only target the element nested within the other element. Ok so the indent doesn't make it nested, one tag actually has to be in the other. In this example I have used this for nav plus the link.

Universal selector

* Star selector that selects everything, you would never use this apart from a 'CSS reset' which he will go through later.

Chapter 3: Colors

Three different types of colours:
  1. Simply typing the color e.g #darkblue
  2. RGB (Red,Green,Black) - each component has a max of 255 and min of 0. So all 0 would be black and all 255 would be white. Whereas 255,0,0 would be red. Also if you add a fourth number that equates to transparancy between 0 and 1.
  3. Hex - based on a number code - easiest way to do this is to look it up on a separate page
  4. If you hover over the color square you can choose from the palette
