Rule Specificity Breakdown

Ranking from least specific to most specific:
  1. Universal Selector (*)
  2. Type Selector (Element Selector) and Pseudo-elements
  3. Class Selector, Attribute Selector, and Pseudo-classes
  4. ID Selector
  5. Inline Styles
  6. !important

Cascading: If two rules are identical, the one at the bottom of the css page will have higher priority because it is the last to be applied.

Specificity: Overrides all other declarations, regardless of their specificity.

Using !important forces the style to be applied, ignoring all other specificity rules. However, !important can be overridden by another !important with higher specificity.

Element Selectors

Elements are like body, p, h1, etc and it automatically selects all those elements. An interesting one is size in px because even with headers it still increases everything proportionately.

Class Selectors

These allow you to get more specific. In CSS you create a class with .[text] {style}, then you can apply to the part of the website you want. So in this example I have applied to the h1 and img within the div.

ID Selectors

This is written in the same way in the stylesheet although with a #. These should rarely if ever be used!! He has stressed it is bad practice to use these and if you do they should only be used once. I get the sense at this stage that you basically want to style everything with class selectors with the occasional element selector.

Chapter 3: Colors

Three different types of colours:
  1. Simply typing the color e.g #darkblue
  2. RGB (Red,Green,Black) - each component has a max of 255 and min of 0. So all 0 would be black and all 255 would be white. Whereas 255,0,0 would be red. Also if you add a fourth number that equates to transparancy between 0 and 1.
  3. Hex - based on a number code - easiest way to do this is to look it up on a separate page
  4. If you hover over the color square you can choose from the palette
Color Palette

HSL (Hue/Saturation/Lightness) - This is if you want to get really advanced with colors.

Make sure you go on Coolors to generate colour palettes and test contrasts which is really important. For some sort of website standards that assign a numerical value that tells you the contrast between the background and the text and they have minimum scores and tests for this.

Chapter 4: Units & Sizes

All browsers have default settings, so if you got into Chrome setting you can choose a font size of large, medium, small. This is known as your root settings, and is usually always at 16px. This is known as your root element, and is important for percentages later as percentages will be relative to this number.

px = pixels, and it the most common size unit. So whilst you can type font size = large, these are essentially pre-set sizes. If you want more customization then pixels is a good choice.

It is not recommended to apply a pixel font size for the entire doc (such as * or html selector) because this then overrides the root settings in your chrome browser which is apparently bad

Percentages

The default setting for elements from a width perspective is to use the entirety of the width that's availabe on the page. So a heading element will take up the entire left-right of the page even if the text only takes up a certain amount. See header border for example.

You can use percetage to reduce this width from the default setting. For example, this paragraph is 50%

If you have a parent, like an article, where the width is set to 50%, and you create a child element, like a section, and set that to 50%, then rule will apply 50% of the 50% (25% of the root)
rem and em

This is supposedly the best way to apply font sizes. From what I understand, rem = relative to root element. So 1rem would keep the text the same size as the default which is 16px. If you put 2rem then it will double the text size to 32pixels.

rem vs em

From what I understand, rem always = relative to root (browswer settings). Whereas em means the parent or the selector. So similar to the percentages where you can do 50% of 50%. Em is bascially when you apply size relative to that element, not the root. This is supposed to be great for padding and buttons.

ch (character)

Another way of formatting, so you can use character settings to define columns e.g. width = 40characters
